# Claude Code 인계 컨텍스트
> 이 파일을 Claude Code 세션 시작 시 첨부하거나 붙여넣어 사용합니다.
> 생성일: 2026-02-20

---

## 1. 현재 작업 목표

**Claude Code 규칙 매뉴얼 작성 프로젝트**

이인석님의 개발 환경 전반에 걸쳐 Claude Code가 일관되게 동작하도록
글로벌 규칙과 프로젝트 템플릿을 문서화하는 작업입니다.

---

## 2. 배경 및 결정 사항

### 사용 환경
- Windows + WSL2 (주 개발환경)
- 원격 서버 SSH

### 규칙 적용 구조
```
~/.claude/CLAUDE.md          ← 글로벌 (모든 프로젝트 공통)
<project-root>/CLAUDE.md     ← 프로젝트별 (글로벌 상속 + 오버라이드)
```

### 핵심 규칙 영역 (논의 확정)
1. **코드 스타일** — C/C++ 임베디드 중심, 네이밍/안전코딩/에러처리 포함
2. **Git 워크플로우** — 단순 Git 탈피, worktree/hooks/자동화 스크립트 포함
3. **임베디드 빌드 환경** — 크로스 컴파일, WSL2/SSH 경로 분기
4. **바이브 코딩 + 지식 학습 모드**:
   - 구현 전 `[설계 의도]` 블록 출력
   - 능동적 대안 제안
   - 기술 부채 `TODO/FIXME/HACK` 명시
   - 구현 완료 후 `docs/impl/` 에 구현 요약 문서 산출

## 3. 현재 생성된 파일

### 파일 A — 글로벌 규칙
```
파일명: global_CLAUDE.md
배포 위치: ~/.claude/CLAUDE.md
```

**포함 섹션:**
- 섹션 1: 환경 감지 및 적응 (WSL2 / SSH 분기)
- 섹션 2: 응답 언어 및 커뮤니케이션
- 섹션 3: 설계 우선 원칙 (바이브 코딩 + 지식 학습) — 3.1~3.4
- 섹션 4: C/C++ 임베디드 코드 스타일 — 4.1~4.5
- 섹션 5: Git 워크플로우 — 자동화 및 확장 — 5.1~5.5
- 섹션 6: 임베디드 빌드 환경 — 6.1~6.3
- 섹션 7: 코드 리뷰 및 산출물 기준 (체크리스트)

### 파일 B — 프로젝트 템플릿
```
파일명: project_CLAUDE.md
배포 위치: <각 프로젝트 루트>/CLAUDE.md
```

**포함 섹션:**
- 프로젝트 개요 (타겟 보드, 툴체인 등 — [placeholder] 채워서 사용)
- 섹션 1: 프로젝트 디렉토리 구조
- 섹션 2: 환경별 설정 오버라이드 (WSL2 / SSH)
- 섹션 3: 레이어 아키텍처 규칙 (APP → HAL → Driver 경계)
- 섹션 4: 프로젝트별 코드 규칙 오버라이드
- 섹션 5: 빌드 명령어
- 섹션 6: Git 워크플로우 — 프로젝트 확장
- 섹션 7: 구현 문서 산출 위치
- 섹션 8: 알려진 환경 이슈 및 해결책 (누적 테이블)

## 4. 수정 작업 방식 (중요)

### 원칙
> 이 프로젝트는 **대화를 통한 수정사항 반영** 방식으로 진행합니다.
> Claude Code가 파일을 직접 임의 편집하지 않습니다.

### 수정 요청 → 반영 흐름
```
1. 사용자가 자연어로 수정 요청
   예: "섹션 4에 MISRA-C 관련 규칙 추가해줘"

2. Claude Code는 변경 내용을 먼저 설명
   - 어느 파일의 어느 섹션을 수정하는지
   - 추가/수정/삭제 내용 요약

3. 사용자 승인 후 파일 수정 실행

4. 수정 후 변경 요약을 이 컨텍스트 파일(5번 섹션)에 기록
```

### Claude Code가 하면 안 되는 것
- 승인 없이 파일 내용 변경
- 섹션 삭제 또는 구조 변경
- 이 컨텍스트 파일 자체를 임의 수정

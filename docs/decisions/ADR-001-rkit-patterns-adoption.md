# ADR-001: rkit 패턴 채택 — 템플릿, Hooks, ADR 도입

- **날짜**: 2026-04-10
- **상태**: **채택**
- **결정자**: inseok + Claude

## 맥락

ai-dev-toolkit은 AI 도구 지침 라이브러리로, 에이전트 파이프라인(6개)과 스킬(6개)을 보유하고 있다.
rkit 플러그인이 더 성숙한 패턴(템플릿 시스템, Hook 자동화, 결정 기록, 품질 게이트)을 가지고 있어
이를 분석하여 적용 가능한 패턴을 채택하기로 했다.

## 검토한 선택지

| 선택지 | 장점 | 단점 |
|--------|------|------|
| A: rkit 패턴 전체 이식 | 완전한 자동화 | JS 런타임 의존, 과도한 복잡도 |
| B: 핵심 패턴만 선택 채택 | 적절한 복잡도, Markdown 기반 유지 | 일부 기능 누락 |
| C: 현상 유지 | 변경 비용 없음 | 성장 정체, 수동 작업 반복 |

## 결정

**B: 핵심 패턴만 선택 채택**

채택 항목:
1. **문서 템플릿** (`.claude/templates/`) — Plan, Design, Report, ADR 4종
2. **PostToolUse Hook** — 지침 파일 줄 수 자동 검사
3. **ADR 시스템** (`docs/decisions/`) — 아키텍처 결정 추적
4. **에이전트 템플릿 참조** — 인라인 산출물 형식 → 템플릿 참조로 전환

비채택 항목:
- JS 런타임 기반 상태 머신 (과도한 복잡도)
- 다국어 트리거 (한국어/영어로 충분)
- 품질 게이트 자동화 (수동 검증으로 충분한 규모)

## 결과

- 에이전트 산출물 일관성 향상 (템플릿 기반)
- 지침 파일 200줄 제한 자동 감시
- 결정 히스토리 추적 가능
- 프로젝트 복잡도 소폭 증가 (templates/, hooks/, decisions/)

AI Dev Toolkit v0.3.0

AI 코딩 어시스턴트의 지침 라이브러리이자 플랫폼 도구.
한 곳에서 관리하고, 스킬로 자동화하고, 양방향으로 동기화한다.

왜 필요한가 빠른 시작 구성 요소 에이전트 커맨드 구조

왜 필요한가

문제
같은 지침을 반복 작성
AI 어시스턴트를 여러 프로젝트에서 쓰면 코딩 표준, Git 규칙 등을 매번 다시 쓰게 된다.
해결
한 곳에서 관리 + 자동 배포
blueprints/에 공유 지식을 모으고, 스킬 명령어로 프로젝트에 자동 복제 + 구체화한다.
장점
양방향 동기화
프로젝트에서 발전한 지침을 흡수하고, 라이브러리 업데이트를 다른 프로젝트에 전파한다.

빠른 시작

1. 클론

git clone <repo-url> ~/ai-dev-toolkit
cd ~/ai-dev-toolkit

2. 워크스테이션 세팅 (최초 1회)

# Claude Code에서 실행
/ai-platform-defconfig

심볼릭 링크, 패키지 확인, MCP 설정 등을 자동 처리합니다.

3. 프로젝트에 적용

/project-init ~/my-project        # AI 설정 골격 복제
/project-configure                 # 대화형으로 기술 스택 구체화

커맨드

슬래시 커맨드로 반복 작업을 자동화합니다. .claude/commands/에 정의.

커맨드역할
/handoff세션 종료 시 HANDOFF.md 갱신 + 커밋 + 푸시
/sync-checkblueprints ↔ rules 동기화 상태 점검
/doc-gen소스 파일의 코드 설명 문서 자동 생성
/review변경사항 코드 리뷰 (보안/성능/가독성)
/decisionADR(Architecture Decision Records) 생성/조회

계층 구조

blueprints/
코딩 표준, Git 워크플로우, 설계 원칙 등 모든 도구가 참조하는 공통 지식의 원본 (Single Source of Truth)
.claude/rules/
blueprints를 Claude Code paths frontmatter 형태로 변환한 파생물. 특정 파일 작업 시에만 조건부 로드
.claude/templates/
에이전트와 커맨드가 참조하는 문서 템플릿. 산출물의 일관성을 보장 (rkit 패턴 채택)

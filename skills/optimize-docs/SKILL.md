# Optimize Docs

이 저장소의 지침 파일들을 스캔하여 200줄 제한을 준수하는지 확인하고,
초과 파일을 의미를 보존하면서 최적화한다.

## 대상 파일

인자로 특정 파일 경로가 주어지면 해당 파일만, 없으면 아래 전체를 스캔한다:

- `skills/*/SKILL.md`
- `blueprints/*.md`
- `claude/global_CLAUDE.md`
- `claude/project_CLAUDE.md`
- `CLAUDE.md` (루트)

## 실행 흐름

1. **스캔** — 대상 파일의 줄 수를 `wc -l`로 확인
2. **보고** — 결과를 테이블로 표시, 200줄 초과 파일 강조
3. **분석** — 초과 파일에 대해 원인 분석:
   - 중복 내용이 있는가
   - 표현을 압축할 수 있는가
   - 섹션을 별도 파일로 분리할 수 있는가
4. **제안** — 최적화 방안을 구체적으로 제시 (변경 전/후 비교)
5. **적용** — 사용자 확인 후 편집 실행

## 최적화 원칙

- **의미 보존 우선** — 핵심 내용을 삭제하지 않는다
- **중복 제거** — 다른 파일과 겹치는 내용은 참조로 대체
- **표현 압축** — 같은 뜻을 더 짧게. 불필요한 수식어, 반복 설명 제거
- **구조 분리** — 독립적인 섹션은 별도 파일로 분리 (blueprints/ 활용)
- **가독성 유지** — 줄 수만 줄이고 읽기 어려워지면 안 된다

## 주의사항

- 파일 수정 전 반드시 사용자 확인
- YAML frontmatter는 줄 수에 포함하되 최적화 대상에서 제외
- 200줄 이하인 파일은 "통과"로 표시하고 건드리지 않는다

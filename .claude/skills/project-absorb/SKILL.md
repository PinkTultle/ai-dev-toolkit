# Project Absorb

배포된 프로젝트에서 수정/추가된 지침을 라이브러리로 역반영한다.
항목별로 채택 여부를 사용자와 대화형으로 결정한다.

## 인자

- `$0` — 소스 프로젝트 경로 (미지정 시 deploy.json에서 목록 제시)

## 이 저장소 경로

```
REPO_DIR=${CLAUDE_SKILL_DIR}/../../..
```

## 사전 조건

- 소스 프로젝트에 `CLAUDE.md`가 존재하고 기반 버전이 기록되어 있어야 한다
- Local 버전이 0보다 커야 한다 (수정 이력이 있어야 흡수 대상)

## 실행 흐름

### 1. 대상 선정

인자가 없으면:
1. `workstations/` 디렉토리에서 현재 워크스테이션의 `<alias>.deploy.json`을 찾는다
2. Local > 0인 프로젝트만 필터링하여 번호 목록 제시
3. 사용자가 선택

### 2. 변경사항 추출

프로젝트의 CLAUDE.md를 라이브러리의 **기반 버전 시점 템플릿**과 비교한다:
1. `git show v<기반버전>:claude/project_CLAUDE.md`로 배포 시점 템플릿 추출
2. 프로젝트 현재 CLAUDE.md와 diff
3. 변경사항을 **항목 단위**로 분류:
   - **placeholder 채움** — `/project-configure`가 채운 값 (흡수 대상 아님)
   - **규칙 수정** — 기존 규칙을 변경한 부분
   - **규칙 추가** — 프로젝트에서 새로 추가한 섹션/규칙
   - **규칙 삭제** — 템플릿에 있던 내용을 제거한 부분

### 3. 항목별 채택 판단

변경사항을 하나씩 보여주고 1문 1답으로 채택 여부를 결정한다:

```
[1/N] 규칙 추가: "DB 스키마 변경은 항상 Large 프리셋 적용"
      위치: 프로젝트 CLAUDE.md 4절

      → 이 규칙을 라이브러리에 흡수할까요?
        (1) blueprints/에 범용 규칙으로 추가
        (2) 프로젝트 고유로 유지 (흡수 안 함)
        (3) 수정 후 흡수
```

각 항목에 대해:
- **흡수** → 적절한 blueprint 또는 템플릿 파일에 반영
- **미채택** → 프로젝트 고유로 유지, 기록만 남김
- **수정 후 흡수** → 사용자가 범용화한 표현으로 수정 후 반영

### 4. 반영

채택된 항목을 라이브러리에 반영한다:
1. 대상 파일 식별 (blueprints/*.md, claude/project_CLAUDE.md 등)
2. 적절한 위치에 내용 추가/수정
3. `global_CLAUDE.md` 동기화 필요 여부 확인

### 5. 결과 정리

```markdown
## 흡수 결과

### 채택 (N건)
- [항목]: [반영 위치]

### 미채택 (N건)
- [항목]: 프로젝트 고유로 유지

### 다음 단계
- 라이브러리 변경 검토 후 커밋
- 버전 증가 (Minor 또는 Patch) 후 태그
- 필요 시 /project-update로 다른 프로젝트에 전파
```

## 주의사항

- placeholder 값(프로젝트명, 경로 등)은 흡수 대상에서 자동 제외
- 프로젝트 고유의 특수 규칙과 범용화 가능한 규칙을 구분하여 제안
- 흡수 후 라이브러리에 자동 커밋하지 않음 — 사용자가 검토 후 커밋
- 흡수 완료 후 `/project-update` 실행을 안내하여 다른 프로젝트에 전파 유도
